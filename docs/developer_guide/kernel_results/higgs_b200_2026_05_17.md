# HIGGS B200 Result, 2026-05-17

## Scope

HIGGS dense MLA 2-bit KV cache and fused dense MLA decode for the active
DeepSeek-V3.2-REAP lane. HIGGS replaces TurboQuant as the dense KV path.

## Environment

- Hardware: NVIDIA B200 (`sm100`)
- Kernel type: direct CUDA JIT scalar codec path
- Profiling: CUDA-event split sweeps and IKP/Nsight attribution
- Privacy: synthetic tensors only

## Accepted Changes

- Default B200 split count from 16 to 32.
- Pair-lane packed-byte broadcast.
- Read-only packed-byte loads.
- Auto split policy with 32/48/56/64/72/80/96/128 split choices where measured.
- Final repeat-confirmed 36-split adjustment for the r16/topk2048 bucket.

## Rejected Candidates

| Candidate | Decision |
| --- | --- |
| Global split-64 default | Rejected because rows=4 and rows=16 guardrail shapes regressed. |
| Blanket 48, 96, or 128 split policies | Rejected outside measured long-topk corners because stage2 overhead dominated. |
| Additional split-neighborhood probes after round 8 | Rejected as noise or not pool-confirmed. |

## Verification

- HIGGS integration and quantization dispatch tests passed in lane validation.
- Final GPU0 integration bundle covering NVFP4, HISA, and HIGGS passed 27 tests
  with 1 skip.

## Limits And Next Work

The current packed 258 B slot layout is scalar codec work, not a tensor-core
tile. Further material speedup needs a new CuTe/CZS tensor-op or staging design
rather than more split-count retuning.
