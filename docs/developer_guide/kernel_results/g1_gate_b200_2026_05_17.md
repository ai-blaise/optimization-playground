# G1 Gate B200 Result, 2026-05-17

## Scope

G1 attention output gating for the DeepSeek-V3.2-REAP deployment lane. The
production path applies `attn_out * sigmoid(gate)` before `o_proj`.

## Environment

- Hardware: NVIDIA B200 (`sm100`)
- Shape: token counts 1, 16, 64, 256, 1024 with hidden size 7168
- Profiling: IKP-imported Nsight traces and CUDA-event microbenchmarks
- Privacy: synthetic tensors only

## Current Result

| Shape | Baseline Full Path | Final Fused Path | Speedup |
| --- | ---: | ---: | ---: |
| 1 token | model-style allocation path | final output-only path | 19.9% faster |
| 16 tokens | model-style allocation path | final output-only path | 20.1% faster |
| 64 tokens | model-style allocation path | final output-only path | 20.7% faster |
| 256 tokens | model-style allocation path | final output-only path | 20.6% faster |
| 1024 tokens | model-style allocation path | final output-only path | 0.1% faster |

IKP measured the 256-token incumbent full path
`g1_gate_cute_kernel_v2` at 3.152 us mean. The accepted output-only path
`g1_gate_cute_fused_kernel_v2` measured 2.894 us mean.

## Accepted Changes

- Add SM100 packaging/runtime dispatch so B200 loads a common-ops artifact with
  `sm_100` cubins.
- Expose `g1_gate_forward_fused`.
- Use the output-only fused op from the DeepSeek G1 model hook when available.

The SM100 packaging fix is deployability, not a throughput claim. The speedup
claim applies to the output-only fused dispatch.

## Rejected Candidates

| Candidate | Decision |
| --- | --- |
| Raise launch threshold to 3M elements | Rejected as noise-scale improvement at 256 tokens and neutral elsewhere. |
| Launch geometry sweep | Rejected because useful block/grid variants tied within launch-floor noise. |
| SM100 cache-streaming load/store hints | Rejected because timings tied or slightly regressed. |
| Fuse into ModelOpt/FlashInfer FP4 activation quantization | Blocked as a separate swizzled-scale quantization contract, not a local G1 patch. |
| Rational sigmoid approximation | Rejected because BF16 output error reached 0.21142578. |

## Verification

- Focused G1 test: `10 passed`.
- Correctness max difference matched the full kernel through 64 tokens, then
  stayed within the existing BF16 tolerance at larger token counts.
- Final integration bundle on GPU1 passed 64 focused tests across
  FlashSampling, G1, GatedNorm, and WarpDecode.

## Limits And Next Work

The remaining local G1 kernel is launch-floor limited at small and medium token
counts. The next plausible win is cross-operator fusion into FP4 activation
quantization before `o_proj`, which needs a separate tensor-op-adjacent
quantization design.
