# LayerSplit B200 Result, 2026-05-17

## Scope

LayerSplit stage-copy and fused materialize kernels for B200 deployment. This
lane is raw device-to-device memory movement, not a tensor-core workload.

## Environment

- Hardware: NVIDIA B200 (`sm100`)
- Kernel type: native CUDA copy kernels
- Benchmarking: CUDA-event harnesses under explicit GPU locks
- Privacy: synthetic tensors only

## Current Result

| Comparison | Baseline | Final | Speedup |
| --- | ---: | ---: | ---: |
| <=116 KiB packaged stage kernel | `Tensor.copy_` | custom stage kernel | 1.058x average |
| 128 KiB threshold cells | 0.800x vs direct `Tensor.copy_` | 1.048x vs direct `Tensor.copy_` | 1.31x relative |
| 128 KiB `cute_ms` | 3.681 us | 2.849 us | 1.292x |
| >128 KiB delegated path | about 0.80x vs direct Python copy | 0.89x-0.93x vs direct Python copy | reduced dispatch loss |
| 512 KiB threshold common matrix | previous threshold | final threshold | 1.083x average, up to 1.150x |
| 192-512 KiB intended cells | previous threshold | final threshold | 1.132x-1.150x |

## Accepted Changes

- Package the existing LayerSplit stage kernel in `sgl-kernel`.
- Raise the small-copy threshold to 128 KiB.
- Use direct-prefix `cudaMemcpyAsync` in the delegated contiguous path.
- Raise the small-copy threshold to 512 KiB after same-GPU back-to-back
  validation.

## Rejected Candidates

| Candidate | Decision |
| --- | --- |
| Python runtime helper dispatch | Rejected because direct `Tensor.copy_` was faster for representative payloads. |
| Dynamic small-copy CTA count | Rejected because fixed broad SM residency was faster despite idle warps. |
| 1 MiB, 2 MiB, 4 MiB, and 8 MiB small-copy thresholds | Rejected because larger transfers belonged on the copy-engine path. |
| Fused materialize CTA-threshold sweep | Rejected because the full matrix tied or regressed. |
| Fused materialize 4-way ILP unroll | Rejected because it averaged 0.954x against the incumbent. |

## Verification

- Every benchmark cell checked active-prefix equality.
- Padding verification checked that inactive rows stayed zero.
- CPU/config integration group including LayerSplit passed 64 tests.

## Limits And Next Work

The accepted kernel is memory movement with no GEMM-shaped tensor-core
opportunity. Further local threshold and loop-shape probes are unlikely to move
performance; future gains need producer/consumer fusion outside this copy
kernel.
