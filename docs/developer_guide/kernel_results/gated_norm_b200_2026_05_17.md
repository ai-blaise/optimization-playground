# GatedNorm B200 Result, 2026-05-17

## Scope

Forward-only BF16 GatedNorm inference for the DeepSeek-V3.2-REAP deployment
lane. The path computes:

```text
z = normed @ w_down.T
gate = sigmoid(silu(z) @ w_up.T)
output = normed * gate
```

## Environment

- Hardware: NVIDIA B200 (`sm100`)
- Hidden size: 7168
- Profiling: CUDA-event dispatch sweeps and IKP/Nsight attribution
- Privacy: synthetic tensors only

## Accepted Changes

- Lower rank-specific GEMM dispatch thresholds where B200 measurements showed
  the fused path was faster.
- Add decode-friendly thresholds for low-rank buckets.
- Add a runtime fallback when `sgl_kernel.gated_norm_cute_forward` is exported
  in Python but the installed shared object does not register the torch op.

## Rejected Candidates

| Candidate | Decision |
| --- | --- |
| More aggressive dispatch to GEMM for tiny decode buckets | Rejected where it stayed at launch floor or regressed. |
| Additional low-rank threshold probes after the accepted floors | Rejected when gains were noise-level or not repeat-confirmed. |
| Treat stale native-op registration as fatal | Rejected for production deployability; fallback is safer and keeps requests live. |

## Verification

- GatedNorm focused tests passed in lane validation.

- Final GPU1 integration bundle passed 64 focused tests.

## Limits And Next Work

The current forward path is saturated by launch and small GEMM costs in the low
rank decode buckets. Larger gains need cross-layer or fused-MMA design work, not
another dispatch-threshold sweep.
